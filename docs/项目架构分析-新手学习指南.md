## 1. 项目概述

### 1.1 项目是什么？

**Vibe Music Server** 是一个音乐平台的后端 API 服务，为音乐客户端和管理端提供数据接口。

```
┌─────────────┐     HTTP API      ┌──────────────────┐
│   客户端     │ ◄───────────────► │  本项目 (后端)     │
│ (Vue/前端)  │                    │                  │
└─────────────┘                    │  - 处理业务逻辑   │
                                   │  - 操作数据库     │
┌─────────────┐                    │  - 提供数据接口   │
│   管理端     │ ◄───────────────► │                  │
│ (管理网站)  │                    └────────┬─────────┘
└─────────────┘                             │
                                          ▼
                              ┌────────────────────────┐
                              │     依赖服务           │
                              │  • MySQL (数据库)     │
                              │  • Redis (缓存)        │
                              │  • MinIO (文件存储)    │
                              └────────────────────────┘
```

### 1.2 为什么需要后端服务？

- **数据持久化**：用户数据、歌曲信息需要永久保存
- **业务逻辑**：处理登录、注册、权限验证等复杂逻辑
- **数据安全**：敏感数据（如密码）不能在前端存储
- **统一接口**：为不同客户端（App、Web、小程序）提供统一接口

### 2.3 技术详解（适合新手）

#### 🔹 Spring Boot 是什么？

Spring Boot 是 Spring 框架的简化版，让开发者可以快速搭建项目。

**传统 Spring 项目的问题：**
- 配置复杂
- 依赖管理麻烦
- 需要大量 XML 配置

**Spring Boot 的改进：**
- 自动配置（convention over configuration）
- starter 依赖（一个依赖包含多个相关库）
- 内嵌服务器（可以直接运行）
- 监控和健康检查

#### 🔹 MyBatis-Plus 是什么？

MyBatis-Plus 是 MyBatis 的增强工具，简化数据库操作。

```java
// 传统 MyBatis：需要写很多 SQL
@Select("SELECT * FROM tb_user WHERE id = #{id}")
User selectById(Long id);

// 使用 MyBatis-Plus：自动生成 SQL
// 继承 IService 接口即可获得基本的 CRUD 方法
public interface IUserService extends IService<User> {
    // getById, save, update, remove 等方法自动可用
}
```

#### 🔹 JWT 是什么？

JWT（JSON Web Token）是一种身份认证方式，用于确认用户身份。

```
JWT 结构：header.payload.signature

┌─────────────┐  ┌─────────────┐  ┌─────────────┐
│    Header   │  │   Payload   │  │  Signature  │
│  (算法信息)  │  │  (用户数据)  │  │  (加密签名)  │
└─────────────┘  └─────────────┘  └─────────────┘
```

登录成功后，服务器生成 JWT 返回给客户端。后续请求带上这个 token，服务器验证后即可识别用户身份。

## 4. MVC 分层架构

本项目采用经典的 **MVC 架构**（不是 ASP.NET MVC，是 Java 常用的分层模式）：

```
┌──────────────────────────────────────────────────────────────┐
│                        请求流程                              │
│                                                              │
│   客户端 ───► Controller ───► Service ───► Mapper ───► 数据库│
│                                                              │
│                ▲         ▲         ▲                        │
│                │         │         │                        │
│                │    业务逻辑    数据库操作                   │
│                │    处理        SQL 执行                     │
│                │                                          │
│                └────── 返回结果层层传递 ──────────────────────
└──────────────────────────────────────────────────────────────┘
```

### 4.1 Controller 层（控制层）

**职责**：接收 HTTP 请求，调用 Service 处理，返回响应给客户端

```java
// 一个典型的 Controller 示例
@RestController          // 表示返回 JSON 数据
@RequestMapping("/user")  // URL 前缀
public class UserController {

    @Autowired
    private IUserService userService;  // 注入服务层

    // 处理登录请求
    @PostMapping("/login")
    public Result login(@RequestBody UserLoginDTO dto) {
        // 调用服务层处理业务
        return userService.login(dto);
    }
}
```

**关键注解**：
- `@RestController` = `@Controller` + `@ResponseBody`，返回 JSON
- `@RequestMapping` 定义 URL 路径
- `@GetMapping` / `@PostMapping` / `@PutMapping` / `@DeleteMapping` 定义 HTTP 方法
- `@RequestBody` 接收 JSON 请求体
- `@RequestParam` 接收 URL 参数

### 4.2 Service 层（业务层）

**职责**：处理业务逻辑，调用 Mapper 操作数据库

```java
// Service 接口
public interface IUserService {
    Result login(UserLoginDTO dto);
}

// Service 实现
@Service  // 标记为服务组件
public class UserServiceImpl implements IUserService {

    @Autowired
    private UserMapper userMapper;

    @Override
    public Result login(UserLoginDTO dto) {
        // 1. 验证参数
        // 2. 查询数据库
        // 3. 验证密码
        // 4. 生成 Token
        // 5. 返回结果
    }
}
```

### 4.3 Mapper 层（数据访问层）

**职责**：操作数据库，执行 SQL

```java
// 继承 BaseMapper 后自动拥有 CRUD 方法
@Mapper  // 标记为 Mapper 组件
public interface UserMapper extends BaseMapper<User> {
    // 一些自定义查询可以在这里定义
}
```

## 6. 数据流与请求流程

### 6.1 完整请求流程

以"用户登录"为例：

```
1. 客户端发送 POST 请求
   POST /user/login
   Content-Type: application/json
   Body: {"username": "zhangsan", "password": "123456"}

2. LoginInterceptor 拦截验证
   - 检查 Token 是否存在
   - （登录接口不需要 Token，跳过验证）

3. UserController 接收请求
   @PostMapping("/login")
   public Result login(@RequestBody UserLoginDTO dto) { ... }

4. 验证参数格式
   - 检查 dto 是否有值
   - 格式是否正确

5. 调用 UserService.login()
   - 查询数据库验证用户名
   - 验证密码
   - 生成 JWT Token
   - 存储 Token 到 Redis

6. 返回 Result
   Result.success(token)

7. 客户端收到响应
   {"code": 0, "message": "操作成功", "data": "eyJhbGci..."}
```

### 6.2 后续请求带 Token

```
客户端请求:
   GET /user/getUserInfo
   Authorization: Bearer eyJhbGci...

服务端处理:
   │
   ▼
LoginInterceptor 拦截
   │
   ▼
从请求头提取 Token
   │
   ▼
Redis 中验证 Token 是否有效
   │
   ▼
解析 Token 获取用户信息
   │
   ▼
存入 ThreadLocal
   │
   ▼
继续处理请求...
```

## 7. 关键设计模式

### 7.1 分层设计模式

每层只负责自己的职责：

```
Controller  ──────►  Service  ──────►  Mapper
  │                    │                    │
  │ 只处理请求          │ 只处理业务          │ 只操作数据库
  │ 和参数校验          │ 逻辑                │
  │                    │                    │
```

### 7.2 DTO / VO 模式

**为什么要区分？**

```
┌─────────┐    ┌─────────┐    ┌─────────┐
│  DTO    │    │ Entity  │    │   VO    │
│(接收)   │    │(存储)   │    │(返回)   │
└─────────┘    └─────────┘    └─────────┘
    │              │              ▲
    │              │              │
  前端传来       数据库表        前端需要
  的数据         结构            的数据
```

**Example**:
- Entity (User): 包含密码字段，存入数据库
- DTO (UserLoginDTO): 只包含用户名和密码
- VO (UserVO): 隐藏密码，只返回安全的用户信息

### 7.3 统一异常处理

```java
// GlobalExceptionHandler.java
@ExceptionHandler(Exception.class)
public Result handleException(Exception e) {
    // 统一处理所有异常
    return Result.error("服务器错误: " + e.getMessage());
}
```

## 9. 学习路径建议

### 9.1 推荐学习顺序

```
第1步: 环境搭建
  ├── 安装 JDK 17
  ├── 安装 IDEA
  ├── 安装 MySQL / Redis / MinIO
  └── 导入项目，配置 application.yml

第2步: 理解项目结构
  ├── 阅读 README.md
  ├── 查看目录结构
  └── 理解各层职责

第3步: 运行一个最简单的接口
  ├── 找到 UserController
  ├── 运行项目
  ├── 发送登录请求
  └── 观察返回结果

第4步: 深入理解一个功能模块
  ├── 选择用户模块（最简单的）
  ├── 跟踪登录请求的完整流程
  ├── 理解 Controller → Service → Mapper 的调用

第5步: 学习其他模块
  ├── 歌曲管理
  ├── 歌单管理
  └── 评论系统

第6步: 尝试自己实现一个小功能
  ├── 添加一个新接口
  ├── 修改现有逻辑
  └── 调试和测试
```

### 9.2 关键文件阅读顺序

对于初学者，建议按以下顺序阅读：

1. **启动入口** → `VibeMusicServerApplication.java`
2. **统一响应** → `Result.java` - 理解返回格式
3. **用户控制器** → `UserController.java` - 理解请求处理
4. **用户服务** → `UserServiceImpl.java` - 理解业务逻辑
5. **JWT 工具** → `JwtUtil.java` - 理解身份认证
6. **拦截器** → `LoginInterceptor.java` - 理解请求拦截

### 9.3 核心概念解释

| 概念 | 简单解释 |
|------|---------|
| `@Autowired` | 自动注入依赖 |
| `IService<T>` | MyBatis-Plus 提供的服务接口 |
| `BaseMapper<T>` | MyBatis-Plus 提供的 Mapper 接口 |
| `ThreadLocal` | 每个线程独立的数据存储 |
| `RedisTemplate` | 操作 Redis 的工具 |
